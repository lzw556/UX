10.1 Definition of containing block

计算一个box的位置和尺寸时,会相对于一个特定的矩形,这个矩形叫作元素的containing block.一个元素的containing block定义如下:

  1. 根元素所在的containing block是一个被称为initial containg block的矩形.尺寸为viewport,被固定在画布开始的位置
  2. 对于其他元素,如果其'position'值为'static' or 'relative',那么其containing block由最近的block container box祖先的content edge形成
  3. 如果元素的'position'值为'fixed',那么其containing block由viewport建立
  4. 如果元素的'position'值为'absolute',那么其containing block由最近的positioned box祖先建立,方式如下:
    1. 如果这个祖先是inline element,containing block就是环绕着为该元素生成的第一个和最后一个行内盒的内边距框的边界框(bounding box)
    2. 否则,containing block由祖先的padding edge形成
    如果没有这样的祖先,其containing block就是initial containing block

containing block所在的矩形可以由content edge形成,也可以由padding edge形成,和元素根据box model生成的矩形并不一样

10.3 Calculating widths and margins

'width','margin-left','margin-right','left','right'这些用来布局的属性,其值取决于元素生成box的type和这些属性值之间的相互影响.原则上,used value和computed value相同,就是把'auto'替换为合适的值,百分比相对于containing block进行计算

10.3.1 Inline,non-replaced elements

'width'属性不适用.computed value是'auto'的margin-left或margin-right的used value为0

10.3.2 Inline,replaced elements

computed value是'auto'的margin-left或margin-right的used value为0.
如果元素的'width'的computed value是'auto',并且具有一个固有宽度,那么该元素的'width'的used value就是这个固有宽度

10.3.3 Block-level,non-replaced elements in normal flow

几个水平属性之间必须遵循如下约束:
'margin-left'+'border-left-width'+'padding-left'+'width'+'padding-right'+'border-right-width'+'margin-right' = width of containing block

在满足等式的前提下,如果'width'的值是'auto',那么其他的'auto'都会被变成0,所以等式中'auto'的绝对长度计算如下:

  • 有且仅有一个'auto'值
    1. 'width'值是'auto',那么'width'的used value由等式计算得出
    2. 'margin-left'值是'auto',那么'margin-left'的used value由等式计算得出
    3. 'margin-right'值是'auto',那么'margin-right'的used value由等式计算得出
  • 有2个'auto'值
    1. 'width'值为'auto','margin-left'值为'auto',则'margin-left'的'auto'值会被变成0,'width'的used value由等式计算得出
    2. 'width'值为'auto','margin-right'值为'auto',则'margin-right'的'auto'值会被变成0,'width'的used value由等式计算得出
    3. 'margin-left'和'margin-right'值都是'auto',则'margin-left'等于'margin-right','auto'的绝对长度由等式计算得出,实际上此时box在其containing block中水平居中
  • 有3个'auto'值
    1. 如果这3个属性值都是'auto',那么'margin-left'和'margin-right'的'auto'值会被变成0,'width'的used value由等式计算得出

如果以上属性的computed value不满足这个等式时,就把这种情况叫作'过度限制'.这时如果'direction'值为'ltr',那么会忽略'margin-right'的specified value并将其计算成满足等式所需的值关于'过度限制',在chrome中测试(测试代码:.outer{width:300px;}.inner{width:500px;})时发现并非如此,自己也没有找到相应资料,故先搁置

10.3.4 Block-level,replaced elements in normal flow

'width'属性的used value由10.3.2 Inline,replaced elements来决定,然后由10.3.3 Block-level,non-replaced elements来决定margin.
换句话说,当'width'值是'auto'时,'width'的used value是元素的固有宽度.margin则根据10.3.3中的等式计算得出

10.3.5 Floating,non-replaced elements

computed value是'auto'的margin-left或margin-right的used value为0.如果'width'的computed value是'auto',那么其used value是'shrink-to-fit' width(自适应宽度)

10.3.6 Floating,replaced elements

computed value是'auto'的margin-left或margin-right的used value为0.'width'属性的used value由10.3.2 Inline,replaced elements来决定.换句话说,当'width'值是'auto'时,'width'的used value是元素的固有宽度

10.3.7 Absolutely positioned,non-replaced elements

几个水平属性之间必须遵循如下约束:
'left'+'margin-left'+'border-left-width'+'padding-left'+'width'+'padding-right'+'border-right-width'+'margin-right'+'right' = width of containing block

  1. 如果'left','width'和'right'都不是'auto',然后判断
    1. 如果'margin-left'和'margin-right'都是'auto',则'margin-left'和'margin-right'相等,然后由等式计算出'margin-left'和'margin-right';如果margin-left计算结果为负数则进入下一步
    2. 如果containing block的'direction'值是'ltr'('rtl'),那么'margin-left'('margin-right')值变为0,然后由等式计算出'margin-right'
    3. 如果'margin-left'和'margin-right'中有一个是'auto',则由等式计算出这个值
  2. 否则,首先把值是'auto'的'margin-left'或'margin-right'变成0;然后判断
    1. 如果'left','width'和'right'都是'auto',
      1. 如果containing block的'direction'值是'ltr',则'left'是其在normal flow中的位置;然后再应用规则G计算出'width'和'right'
      2. 如果containing block的'direction'值是'rlt',则'right'是其在normal flow中的位置;然后再应用规则E计算出'width'和'left'
    2. 如果'left'是'auto','width'和'right'都不是'auto',则由等式计算出'left'
    3. 如果'width'是'auto','left'和'right'都不是'auto',则由等式计算出'width'
    4. 如果'right'是'auto','left'和'width'都不是'auto',则由等式计算出'right'
    5. 如果'left'和'width'都是'auto','right'不是'auto',则'width'是'shrink-to-fit'(自适应宽度),然后由等式计算出'left'
    6. 如果'left'和'right'都是'auto','width'不是'auto',然后判断
      1. 如果containing block的'direction'值是'ltr',则'left'是其在normal flow中的位置;然后由等式计算出'right'
      2. 如果containing block的'direction'值是'rtl',则'right'是其在normal flow中的位置;然后由等式计算出'left'
    7. 如果'width'和'right'都是'auto','left不是'auto',则'width'是'shrink-to-fit'(自适应宽度),然后由等式计算出'right'

10.3.8 Absolutely positioned,replaced elements

'width'的used value由10.3.2 Inline,replaced elements决定.

  1. 如果'left'和'right'都是'auto',则判断如果containing block的'direction'值是'ltr'('rtl'),则'left'('right')是其在normal flow中的位置;
  2. 如果'left'或'right'是'auto',则把值是'auto'的'margin-left'或'margin-right'变成0,然后由10.3.7中的等式计算出另一个'auto'值
  3. 如果'left'和'right'都不是'auto',
    1. 如果'margin-left'和'margin-right'都是'auto',则'margin-left'和'margin-right'相等且其值不是负数,然后由10.3.7中的等式计算出'margin-left'和'margin-right'
    2. 否则,如果'margin-left'和'margin-right'相等时由等式计算出其值是负数,则如果containing block的'direction'值是'ltr'('rtl'),那么'margin-left'('margin-right')值变为0,然后由等式计算出'margin-right'
    3. 如果'margin-left'和'margin-right'中有一个是'auto',则由10.3.7中的等式计算出这个值

10.3.9 'Inline-block',non-replaced elements in normal flow

如果'width'是'auto',其used value就是'shrink-to-fit'(自适应宽度).computed value为'auto'的'margin-left'或'margin-right'的used value变为'0'.(实际上和10.3.5 Floating, non-replaced elements中的规则是一样的)

10.3.10 'Inline-block',replaced elements in normal flow

和10.3.2 Inline,replaced elements完全一致

  3. 如果这个初步宽度小于'min-width',就再应用一遍上面的规则,但这次用'min-width'的值作为'width'的computed value

10.6 Calculating heights and margins

10.6.1 Inline,non-replaced elements

'height'属性不适用.content area的height应该基于font,但本规范没有具体说明.例如,UA可能会使用font的em-box或font的ascender和descender(前者能够确保编写者可以相对于'line-height'控制背景样式化，但会导致字形绘制在content area外;后者能够确保em-box上方和下方的字形仍然在content area内,但会导致不同的font有不同大小的box);
'margin-top'和'margin-bottom'属性适用但无效果

inline,non-replaced element垂直方向上的padding,border和margin从content area的顶部和底部开始,和'line-height'无关.'line-height'只在计算line box的height时使用

10.6.2 Inline replaced elements,block-level replaced elements in normal flow,'inline-block' replaced elements in normal flow and floating replaced elements

computed value是'auto'的margin-top或margin-bottom的used value为0.如果元素的'height'的computed value是'auto',并且具有一个固有高度,那么该元素的'height'的used value就是这个固有高度

10.6.3 Block-level non-replaced elements in normal flow when 'overflow' computes to 'visible'

computed value是'auto'的margin-top或margin-bottom的used value为0.如果'height'是'auto',元素的高度取决于其是否含有block-level子级以及该子级是否拥有padding or border:

元素的高度是从top content edge到下列位置之间的距离:

  1. 最后一个line box的bottom edge,如果该box建立了一个inline formatting context的话
  2. 最后一个in-flow子元素的margin-bottom edge,如果该子元素的margin-bottom没有和元素的margin-bottom发生collapse
  3. 最后一个in-flow子元素的border-bottom edge,如果该子元素的margin-top没有和元素的margin-bottom发生collapse
  4. 0

只考虑normal flow中的子元素

10.6.4 Absolutely positioned,non-replaced elements

几个垂直属性之间必须遵循如下约束:
'top'+'margin-top'+'border-top-width'+'padding-top'+'height'+'padding-bottom'+'border-bottom-width'+'margin-bottom'+'bottom' = height of containing block

  1. 如果'top','height'和'bottom'都不是'auto',然后判断
    1. 如果'margin-top'和'margin-bottom'都是'auto',则'margin-top'和'margin-bottom'相等,然后由等式计算出'margin-top'和'margin-bottom'
    2. 如果'margin-left'和'margin-right'中有一个是'auto',则由等式计算出这个值
  2. 否则,首先把值是'auto'的'margin-top'或'margin-bottom'变成0;然后判断
    1. 如果'top','height'和'bottom'都是'auto',则'top'是其在normal flow中的位置;然后再应用规则D计算出'height'和'bottom'
    2. 如果'top'和'height'是'auto','bottom'不是'auto',则'height'由10.6.7中规则计算得出,然后由等式计算出'top'
    3. 如果'top'和'bottom'是'auto','height'不是'auto',则'top'是其在normal flow中的位置;然后由等式计算出'bottom'
    4. 如果'height'和'bottom'是'auto','top'不是'auto',则'height'由10.6.7中规则计算得出,然后由等式计算出'bottom'
    5. 如果'top'是'auto','height'和'bottom'都不是'auto',则由等式计算出'top'
    6. 如果'height'是'auto','top'和'bottom'都不是'auto',则由等式计算出'height'
    7. 如果'bottom'是'auto','top'和'height'都不是'auto',则由等式计算出'bottom'

10.6.5 Absolutely positioned,replaced elements

'height'由10.6.2 inline replaced elements决定

  1. 如果'top'和'bottom'都是'auto',则'top'是其在normal flow中的位置
  2. 如果'bottom'是'auto',则把值是'auto'的'margin-top'或'margin-bottom'变成0
  3. 如果'top'和'bottom'都不是'auto',则判断如果'margin-top'和'margin-bottom'都是'auto',则上面的约束等式中'margin-top'和'margin-bottom'值相等
  4. 如果此时只剩下一个'auto'了，由等式求出这个值

10.6.6 Complicated cases

本节适用于:

  • block-level,non-replaced elements in normal flow,当 'overflow'不是'visible'时
  • inline-block,non-replaced elements
  • floating,non-replaced elements

computed value是'auto'的margin-top或margin-bottom的used value为0.
如果'height'是'auto','height'由10.6.7中规则中得出

对于'inline-block'元素,计算line box高度时使用margin box

10.6.7 'Auto' heights for block formatting context roots

在特定情况下(例如,上面的10.6.4和10.6.6),一个建立了block formatting context的元素的'height'计算如下:

如果元素中只含有inline-level子元素,'height'就是最高line box顶部和最低line box底部之间的距离

如果元素中含有block-level子元素,'height'就是最高block-level子级box的margin-top edge和最低block-level子级box的margin-bottom edge之间的距离

绝对定位的子元素会被忽略,相对定位的子元素不考虑其偏移

另外,如果元素中含有浮动后代并且这个浮动后代的margin-bottom edge在元素的bottom content edge下方,那么'height'会增加以至于包含这些edge

10.8 Line height calculations:the 'line-height' and 'vertical-align' properties

line box的height由以下规则决定:

  1. 计算line box中每一个inline-level box的height(此height不是box的content area的height).对于替换元素,inline-block元素和inline-table元素,这个height是其margin box的height;
    对于inline box,这个height就是其line-height
  2. inline-level box是根据其'vertical-align'属性进行垂直对齐的.
  3. line box的height就是最高box的顶部到最低box的底部之间的距离

空的inline元素生成空的inline box,但是这些box仍然具有margin,padding,border和line-height,从而影响这些计算,就像有内容的inline元素一样

10.8.1 Leading and half-leading

CSS假定任何字体都具有字体度量(font metric),其指定了每个字体特有的baseline上方的height和baseline下方的depth(深度).在这一章中对于一个给定大小的指定字体,使用A表示height,使用D表示depth,并且定义AD=A+D

对于每一个glyph(字形),UA会决定要添加的Leading(L),L='line-height'-AD(其中L可能为负).Leading被等分为2,一半被添加在A的上方,一半被添加在D的下方

inline box在line box中的height包含所有字形及上下2个半Leading,所以刚好是'line-height'

尽管inline,non-replaced元素(会生成inline box)的margin,border和padding不会计入line box height的计算,但是仍然会被渲染.这意味着如果'line-height'小于content area的高度,padding和border的背景和颜色可能会流入相邻的line box中

line-height
Value:normal | <number> | <length> | <percentage> | inherit
Initial:normal
Applies to:all elements
Inheritedyes
Percentages:参考元素自身的font-size
Media:
Computed value:对于<length>和<percentage>是绝对值,否则和specified value相同

对于一个由inline-level元素组成的block container box,'line-height'指定了line box的最小高度(个人理解,最小高度就是说当这个block container box中包含的inline-level元素全部是inline box时每一个line box的height;如果 其中一个line box中包含一个height比'line-height'大的图片,此时元素的'line-height'就会重新计算了(会比指定的'line-height'大)),就好像每一个line box以一个具有和该元素相同的font-size和line-height且width为0的inline box开始一样,我们把这种假想盒叫做"strut"

对于non-replaced inline元素,'line-height'指定的height被用来计算line box的height

normal
告诉UA根据元素的font设置一个合理的used value,我们推荐应介于1.0到1.2之间.和<number>含义一样,computed value是'normal'
<length>
指定长度被用来计算line box的height.负值是非法的
<number>
对应的used value是number*元素自身的font-size.负值是非法的.computed value和specified value相同
<percentage>
对应的computed value是percentage*元素自身的computed font-size.负值是非法的

当一个block container box中所有的inline box都只有一个'line-height'并且具有相同的font(inline-level box中不存在replaced元素,inline-block元素),那么2个连续line box的baseline之间的距离刚好是'line-height'

vertical-align
Value:baseline | sub | super | top | bottom | text-top | text-bottom | middle | <percentage> | <length> | inherit
Initial:baseline
Applies to:inline-level and 'table-cell' elements
Inheritedno
Percentages:参考元素自身的line-height

该属性会影响一个inline-level box在line box中的垂直定位

该属性在table上下文中具有不同的意义,详见'vertical-algin' for 'table-cell' element

下面的值只对父级inline元素或一个父级block container element中的strut有效(????????????)

在下面的定义中,对于inline,non-replaced元素,用于对齐的box就是height是'line-height'的box(包含box的字形和上下方的半Leading);
对于其他元素,用于对齐的box是margin box

baseline
把box的baseline和其父级box的baseline对齐.如果box没有baseline,把box的margin-bottom edge和其父级box的baseline对齐
sub
把box的baseline降低到合适的位置作为父级box的下标(该值不影响该元素文本的font-size)
super
把box的baseline提升到合适的位置作为父级box的上标(该值不影响该元素文本的font-size)
text-top
把box的顶部和其父级box的content area的顶部对齐
text-bottom
把box的底部和其父级box的content area的底部对齐
middle
把box的垂直中点和其父级box的baseline加上父级的x-height的一半对齐
<percentage>
把box提升(正值)或降低(负值)指定距离(这个距离值='line-height'*percentage)
<length>
把box提升(正值)或降低(负值)指定距离

下面的值让元素相对于一个line box对齐

top
把对齐子树顶端和line box的顶端对齐
bottom
把对齐子树底端和line box的底端对齐
