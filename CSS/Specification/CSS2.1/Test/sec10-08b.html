对于replaced element,inline-block element来说, 在参与计算其所在line box的height时，使用的有效height是其margin box的height

replaced inline-block element

对于non-replaced inline element,在参与计算其所在line box的height时，使用的有效height是 half heading + the height of content-area + half heading = 'line-height'

primary 0 + 31px + 0
primary -5px + 31px + -5px
primary 5px + 31px + 5px

在上面3个例子中,因为strut和primary的font-size不同,故两者的baseline也不同,line-height不同.为了避免因此造成的干扰,设置strut和primary的line-height相同,primary的vertical-align:top

对于non-replaced inline element,其padding, border和margin不参与计算line box的height

primary 0 + 31px + 0

偶然发现当half heading不是整数时,每个浏览器表现不一致,以下是half heading是2.5px时的情况:

在chrome下:

primary 2px + 31px + 3px

在firefox下:

primary 3px + 31px + 2px

所以在设置line-height时最好让half heading为整数
