6.1 Specified,computed,and actual values

一旦UA已经解析了文档并构造文档树,它就必须为树中的每一个元素的每一个css属性(适用于目标媒体类型)赋值

css属性的最终值是通过4步计算得到的:首先通过规范确定specified value,然后把specified value进一步处理为computed value(主要用于后代继承),然后
如果有必要则把computed value转化为一个绝对值即used value,最后根据本地环境限制进行转换即actual value

6.1.1 Specified values

UA必须先根据下面的机制(按优先顺序)给每一个css属性分配一个specified value:

  1. 如果cascade产生一个值,就使用它
  2. 否则,如果这个属性是默认可以被继承的并且元素不是文档树的根节点,就使用其父元素的computed value
  3. 否则,使用这个属性的initial value

6.1.2 Computed values

在层叠期间specified valued被处理为computed value,例如URL被处理为绝对路径,em和ex单位被计算成px或绝对长度.计算一个属性值不需要UA渲染文档

属性的computed value由属性定义中的computed value指定

6.1.3 Used values

处理computed value时,尽量不要格式化文档.但是有些属性值只能在文档布局完成时才被确定.例如,如果一个元素的width
是其包含块的特定百分比,那么在其包含块的宽度确定之前这个width都无法确定.used value是把computed value剩余的依赖
都处理后得到的绝对值.

6.1.4 Actual values

原则上,used value被用于渲染,但是UA可能无法在给定的环境中使用该值.例如,UA可能只能用整数像素width来渲染border,因此不得不对宽度进行近似处理.
actual value就是对used value进行近似处理后的值.

6.2 Inheritance

每一个css属性,其定义中都指出了该属性是默认可以被继承的还是默认不可以被继承的(inherited:yes/no).
当元素的一个继承属性(inherited:yes)没有指定值时(6.1.1),就会取其父元素中相同属性的computed value.这个值会成为子元素的specified value和computed value.
当元素的一个非继承属性(inherited:no)没有指定值是(6.1.1),就会取自身的initial value

6.2.1 The 'inherit' value

对于一个给定元素,inherit表示属性值使用其父元素中相同属性的computed value,不管此属性是否默认可以被继承.

6.4 The cascade

样式表有3种origin:author,user and UA

  • Author.就是开发人员写的样式表
  • User:就是浏览器使用者指定的样式表文件(详见书签)
  • UA:就是浏览器默认样式表

这3种来源的样式表在一定范围内会发生重叠,并且根据层叠相互影响

css cascade给每一条样式规则设置了权重,当同一元素应用几条规则时,权重最大的优先

6.4.1 Cascading order

  1. 找出匹配到的所有declaration
  2. 根据importance(normal和important)和origin排序,优先级从上到下依次升高
    1. UA declaration
    2. user normal declaration
    3. author normal declaration
    4. author important declaration
    5. user important declaration
  3. 如果importance和origin相同,则根据selector的specificity来排序:更特殊的selector会重写一般的selector,伪元素和伪类分别算作常规元素和常规类
  4. 最后,根据指定顺序排序:如果2个declaration的importance,origin和specificity都相同,后指定的生效

6.4.3 Calculating a selector's specificity

  • style属性中的declaration(=a),如果有则a=1,否则a=0
  • 计算selector中的ID属性的数量(=b),如果有n个ID则b=n,没有则b=0
  • 计算selector中的其他属性(类选择器本质上也是属性选择器)和伪类的数量(=c)
  • 计算selector中的元素名和伪元素的数量(=d)

a-b-c-d4种数量连起来表示selector的specificity

*{}                        /* specificity=0,0,0,0 */
p:befor{}                  /* specificity=0,0,0,2 */
.container a[href]:hover{} /* specificity=0,0,3,1 */
#id{}                      /* specificity=0,1,0,0 */
style=""                   /* specificity=1,0,0,0 */
