4.2.5 The meta element

Categories
Metadata content
如果itemprop特性存在, Flow content or Phrasing content
允许的父元素类型
作为head元素的子元素, eg. <meta charset="UTF-8">
作为head元素的子元素, eg. <meta name="keywords" content="html spec">
作为head元素的子元素, eg. <meta http-equiv="X-UA-Compatible" content="ie=edge">
允许的子元素类型
nothing
Content attributes:
Global attributes
name-metadata
http-equiv-编译指令
content-当meta元素存在特性http-equiv或name时, meta元素的值
charset-字符编码声明
DOM interface
interface HTMLMetaElement : HTMLElement {
    [HTMLConstructor] constructor();

    [CEReactions] attribute DOMString name;
    [CEReactions] attribute DOMString httpEquiv;
    [CEReactions] attribute DOMString content;
}
4.2.5.1 Standard metadata names
application-name
定义正运行在该网页上的网络应用名称
author
文档的作者
keywords
包含与逗号分隔的页面内容相关的单词
description
其中包含页面内容的简短和精确的描述, 一些浏览器将其作为书签的默认描述
generator
包含生成页面的软件的标识符
referrer
控制所有从该文档发出的 HTTP 请求中HTTP Referer 首部的内容
theme-color
viewport(不是规范)
4.2.5.3 Pragma directives

当meta元素被指定http-equiv特性时, 这个元素是一个编译指令

http-equiv特性值如下:

content-language
content-type
eg. <meta http-equiv="content-type" content="text/html;charset=utf-8">
default-style
refresh
eg. <meta http-equiv="Refresh" content="300"> 间隔5分钟刷新一次
eg. <meta http-equiv="Refresh" content="30; url=http://www.baidu.com"> 30秒后重定向至新的url
content-security-policy
4.2.5.4 Specifying the document's character encoding

eg. eg. <meta charset="utf-8">

4.3.3 The section element

section元素表示文档中一个带标题的内容分组, 没有更具体的语义, 通常包含一个标题

Categories
Flow content

Flow content
允许的子元素类型
Flow content
Content attributes:
Global attributes
DOM interface
使用 HTMLElement

用法:

  • 语义中强调是一个内容分组, 即多个分组之一, 本身无法构成独立的内容,需要所有相关并列分组共同组成一个独立的整体
  • 一般都包含一个标题(h1-h6)
  • 不要把section当作通用容器元素来使用, 比如当一个元素的片段仅需要美化的时候, 此时可以用div来替代
